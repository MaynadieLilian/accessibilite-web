Attaques CSRF

Publié le 25 janvier 2026 par Lilian

Une attaque CSRF (Cross-Site Request Forgery) force un utilisateur authentifié à exécuter des actions non désirées sur une application web dans laquelle il est actuellement authentifié. L'attaquant exploite la confiance qu'une application web a envers le navigateur de l'utilisateur.

Types et vecteurs d'attaques CSRF
Type/Vecteur Description Exemples d'actions
GET-based CSRF Exploite les requêtes GET pour effectuer des actions (mauvaise pratique du site) Transfert d'argent, modification de paramètres
POST-based CSRF Utilise un formulaire caché qui soumet automatiquement une requête POST Changement de mot de passe, suppression de compte
JSON-based CSRF Cible les API REST qui acceptent des requêtes JSON sans validation CSRF Modification de données via API
Login CSRF Force l'utilisateur à se connecter avec les identifiants de l'attaquant Collecte d'informations sensibles
Source : OWASP - Cross-Site Request Forgery

Prévention

Pour se protéger contre les attaques CSRF, il est essentiel d'utiliser des tokens CSRF uniques pour chaque session et requête, d'implémenter le SameSite cookie attribute, de vérifier l'en-tête Origin/Referer, et d'utiliser l'authentification multi-facteurs pour les actions sensibles.
