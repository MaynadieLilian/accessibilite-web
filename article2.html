Cross-Site Scripting (XSS)

Publié le 18 janvier 2026 par Lilian

Le Cross-Site Scripting (XSS) est une vulnérabilité de sécurité qui permet à un attaquant d'injecter du code malveillant (généralement JavaScript) dans des pages web vues par d'autres utilisateurs. Cette attaque exploite la confiance qu'un utilisateur a envers un site web particulier.

Types d'attaques XSS

Types de XSS courantes et leurs impacts
Type d'attaque Description Impact
XSS Réfléchi (Reflected) Le script malveillant est immédiatement renvoyé par le serveur dans la réponse HTTP Vol de session, redirection malveillante
XSS Stocké (Stored) Le script est stocké de façon permanente sur le serveur (base de données, forum, commentaire) Compromission massive d'utilisateurs
XSS DOM-based La vulnérabilité se trouve dans le code côté client, le payload n'est jamais envoyé au serveur Manipulation du DOM, vol d'informations
XSS Blind Le payload XSS est exécuté dans un contexte où l'attaquant ne voit pas directement le résultat Exfiltration de données d'administration
Source : OWASP - Cross Site Scripting

Prévention

Pour se protéger contre les attaques XSS, il est essentiel d'encoder correctement les sorties HTML, d'utiliser une Content Security Policy (CSP), de valider et nettoyer toutes les entrées utilisateur, et d'utiliser des frameworks modernes qui échappent automatiquement les données.
